Live Stream Policy

Importance (stored as sublevel on the record) is which humans an event reaches (high: everyone, including digests and email; normal: live-page watchers; low: deliberately verbose viewers only). Channels select live events at or above an importance threshold. It is set in the event's code and changed by changing the event. Live is a separate category — "interesting to some humans, now": each event declares a live default, and the override (stored in SysConfig) is the runtime attention knob that wins when set. The live stream view shows the events whose effective state is live.
